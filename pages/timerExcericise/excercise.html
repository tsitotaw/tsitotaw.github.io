  4. What is "Unobtrusive Javascript"? What is the practical application of Unobtrusive Javascript (and the reasons for using it)?

    Unobtrusive Javascript is kind of a design philosophy and opinioin, which guide us to separate our HTML structure and JS behaviors - doing that will help us implement separation of concerns and keep our codes clean. by removing the embedded script logic and the event handlers from HTML, we made sure our HTML page will work regardless of any external dependency - in case browsers donot support JS, or JS is disabled, or make the site's content accessible for visually impaired or search engines or any other reason, our app will work in its basic form. But then, if everything is in place, we can add the additional behavior using our JS, when allowed, which inturn will use the global document object to put its magic. Also, by moving JS out, we can help developers only focus on specific technology and also easily introduce caching, minifing, uglifing, concatinating and all other techniques and boost the performance of our app
